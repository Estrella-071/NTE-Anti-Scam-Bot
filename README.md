# 巡哨鼠鼠 (Officer Whisker)

Discord 伺服器的自動化防洗版機器人。  
分析發言頻率與跨頻道行為來攔截惡意訊息，自動處置後通報給管理員覆核。

## 偵測機制

### 快速洗版（漏桶演算法）

每位使用者有一個獨立的漏桶，容量上限 100。水位以每秒 5 的速率持續流失。

每則訊息會往桶裡加水：

| 條件 | 水位增量 |
|------|---------|
| 基礎值 | +10 |
| 內容跟上一則一樣 | 額外 +35（合計 +45） |
| 訊息長度不到 5 字元 | 減免 -5（合計 +5） |

水位到 100 就判定洗版。舉例來說，連貼 3 則一樣的內容 = 45 × 3 = 135，不到 2 秒就會觸發；如果每則都不同，大約要連發 11 則才會滿。

水位歸零後歷史紀錄會跟著清掉，不會拿很久以前的對話來誤判。另外每 10 分鐘會跑一次全域清掃，釋放已經沒在活動的桶。

如果洗版者刻意每則都發不一樣的內容想規避（重複內容不到 3 則、但總共發了 4 則以上），系統會改擷取最後 5 則不重複的訊息當作通報樣本。

### IME 去重

注音、倉頡、拼音這類輸入法按 Enter 確認時，Discord 偶爾會重送同一則訊息。為避免誤判，同一人在 500ms 內送出一模一樣的內容會被跳過，不計入水位。

### 慢速跨頻道洗版

追蹤每個人發送內容的 hash，記錄它出現在哪些頻道，指紋有效期 1 小時。

判定規則（依優先順序）：

1. 短訊息豁免 — 內容不到 10 字元不算，避免「哈囉」「好喔」這種日常短語被抓。
2. API 異常 — 1 秒內在兩個不同頻道出現相同內容，直接判定是用 API 發的，升到 4 星。
3. 累積門檻 — 同一內容散佈到 3 個以上頻道就觸發。

### 累進處分（通緝值）

每次觸發都會累加星數，對應不同處罰：

* 0 → 1 ⭐：刪除該波訊息
* 1 → 2 ⭐：刪除 + 禁言 5 分鐘
* 2 → 3 ⭐：刪除 + 禁言 15 分鐘
* 3 → 4 ⭐：刪除 + 禁言 1 小時
* 跨頻道或 API 偵測直接跳到 4 ⭐：刪除 + 禁言 1 天，等人工審核

每 12 小時跑一次排程，連續 7 天沒再犯就自動扣 1 星。

### 通報與覆核

觸發後會在指定頻道發通報，包含違規者、觸發頻道、原因和擷取內容（惡意連結會自動遮蔽）。

通報下方有操作按鈕，管理員或擁有「管理成員」權限的人可以選：
* 解除禁言 — 誤判的話一鍵解除

處置完成後按鈕會收回，結果跟操作者 ID 會更新到通報內容裡。

## 管理設定

### /setup

管理員執行 `/setup` 開啟設定面板：
* 通報頻道 — 選擇接收通報的文字頻道
* 身分組白名單 — 選擇不受偵測的身分組
* 成員白名單 — 選擇不受偵測的個別使用者

### /reset

管理員執行 `/reset @使用者` 可以把指定使用者的通緝值（星數）直接歸零。適用於誤判或決定給對方一次機會的情況。

## 技術棧

* TypeScript / Node.js
* Discord.js v14
* [Turso](https://turso.tech)（libSQL 雲端 SQLite）— 儲存通緝積分和伺服器設定，重啟不會掉資料
* [Render](https://render.com) — 雲端部署，推 main 就自動更新

## 部署

### 環境變數

把 `.env.example` 改名成 `.env`，填入：

| 變數 | 說明 |
|-----|------|
| `BOT_TOKEN` | Discord Bot Token |
| `TURSO_URL` | Turso 連線 URL（`libsql://xxx.turso.io`） |
| `TURSO_AUTH_TOKEN` | Turso 認證 Token |

### Render

1. 在 Render 建 Web Service，綁這個 repo
2. Build Command: `npm install && npm run build`
3. Start Command: `node dist/index.js`
4. 在 Environment 加上面三個環境變數
5. 之後推 `main` 就會自動部署
